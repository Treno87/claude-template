# Project Guidelines

## Core Principles

- **TDD**: 테스트 먼저 작성 (@tdd.md 참조)
- **Clean Code**: @clean-code.md 참조
- **Tidying**: 리팩토링은 별도 커밋으로 분리

## Tech Stack

[프로젝트별로 수정하세요]

예시:
- Runtime: Node.js / Bun
- Language: TypeScript
- Formatter/Linter: Biome
- Test: Vitest / Jest

## Commands

- `/review`: 코드 리뷰 요청
- `/refactor`: 리팩토링 제안
- `/test`: 테스트 실행

## Rules

1. **한국어로 응답**
2. **변경 전 반드시 설명**
3. **커밋 메시지는 명확하게**
4. **불필요한 파일 생성 금지**

## Workflow

1. 요구사항 분석
2. 테스트 작성 (TDD)
3. 구현
4. 리팩토링 (별도 커밋)
5. 문서화 (필요시)
